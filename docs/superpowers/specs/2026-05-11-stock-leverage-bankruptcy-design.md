# 주식 레버리지 파산채무 설계

## 배경

현재 주식 레버리지는 포지션에 `notional`과 `debt`를 저장하지만, 청산 시 유저 잔고에 실제 마이너스 골드를 남기지는 않는다. 손실은 증거금 한도에서 끝나며, 수익은 포지션 청산 시 골드로 지급된다. 100배 레버리지의 도박성은 유지하되, 통합 골드 경제에서 무위험 고배율 반복 도박처럼 느껴지지 않도록 청산 후 부담을 추가한다.

## 목표

- 통합 골드 구조는 유지한다.
- 실제 마이너스 잔고는 만들지 않는다.
- 100배 레버리지에 “빚맛”과 후속 리스크를 부여한다.
- 유저가 완전히 접을 만큼 막히지 않게, 미래 수익 일부로 자동 상환되게 한다.
- 파산채무가 있으면 새 레버리지 진입만 막고, 일반 주식/RPG/검강화/카지노 등 기본 플레이는 유지한다.

## 제외 범위

- 전역 마이너스 골드 잔고.
- 이자, 연체료, 강제 아이템 압류.
- 일반 주식 매수/매도 제한.
- 기존 주식 가격 생성 로직 변경.

## 제안 설계

### 1. 파산채무 모델

레버리지 포지션이 청산되면 기존처럼 증거금은 전액 손실 처리하고, 추가로 파산채무를 누적한다. 채무는 주식 사용자 상태에 저장하되, 통합 골드 수익 회수를 위해 경제 프로필에서도 접근 가능해야 한다.

필드 예시:

- `bankruptcyDebt`: 남은 파산채무.
- `bankruptcyDebtPaid`: 누적 상환액.
- `bankruptcyCount`: 파산 횟수.
- `lastBankruptcyAt`: 마지막 파산 시각.

### 2. 청산 패널티

청산 패널티는 포지션 증거금과 레버리지 배율로 계산한다.

| 레버리지 | 파산채무 비율 |
| --- | ---: |
| 1~10배 | 증거금의 5% |
| 11~30배 | 증거금의 10% |
| 31~70배 | 증거금의 20% |
| 71~100배 | 증거금의 35% |

예: 100배에 10,000골드 증거금을 넣고 청산되면 증거금 10,000골드를 잃고, 파산채무 3,500골드가 추가된다.

### 3. 자동 상환

파산채무가 있으면 이후 골드 수익의 25%를 자동 상환한다. 상환액은 `min(floor(income * 0.25), bankruptcyDebt)`로 계산한다. 수익이 1골드처럼 너무 작을 경우에는 0골드 상환이 가능하며, 원 수익보다 더 많이 차감하지 않는다.

회수 대상은 “유저가 새로 받는 골드”다. 대표적으로 출석, 미션, RPG 보상, 검 판매, 주식 매도/레버리지 청산 수익, 복권 당첨금, 카지노 지급금, 레벨업 보상이 포함된다.

### 4. 제한 정책

파산채무가 남아 있으면 새 레버리지 포지션 진입을 막는다. 일반 주식 매수/매도, 보유 주식 매도, 일반 게임 플레이는 막지 않는다. 이미 열려 있는 기존 레버리지 포지션은 기존 로직대로 평가/청산한다.

### 5. 유저 표시

주식 관련 응답에는 파산채무가 있는 경우 짧게 표시한다.

- 남은 채무.
- 이번 보상에서 자동 상환된 금액.
- 새 레버리지 진입 제한 여부.

별도 조회 명령은 `/주식 파산상태` 또는 기존 주식 요약 화면 내 섹션으로 노출한다.

## 데이터 흐름

1. 레버리지 포지션 평가 중 equity가 0 이하가 되면 청산으로 판단한다.
2. 기존 손실 `-margin`은 유지한다.
3. 레버리지 배율에 맞는 파산채무를 계산해 누적한다.
4. 이후 골드 지급 경로에서 지급액 일부를 채무 상환으로 돌린다.
5. 지급 결과에는 원래 보상, 상환액, 실제 수령액을 함께 반환한다.
6. 남은 채무가 0이 되면 레버리지 진입 제한이 자동 해제된다.

## 구현 경계

- 주식 시스템은 파산채무 생성과 레버리지 진입 제한을 담당한다.
- 골드 지급 유틸은 수익 자동 상환을 담당한다.
- 직접 `profile.balance += amount`로 지급하는 기존 경로는 회수 누락이 생기지 않도록 공통 헬퍼로 정리한다.
- 기존 통합 골드 정책은 변경하지 않는다.

## 오류 처리

- 채무 값이 음수/NaN이면 0으로 정규화한다.
- 파산채무가 있는 유저가 레버리지 진입을 시도하면 남은 채무와 상환 방법을 알려주는 에러를 낸다.
- 상환 계산 중 지급액이 0이면 상환도 0으로 처리한다.
- 기존 저장 데이터에 파산채무 필드가 없어도 기본값 0으로 동작한다.

## 테스트 전략

- 100배 청산 시 증거금의 35%가 파산채무로 누적되는지 검증한다.
- 10배/30배/70배/100배 경계값의 패널티 구간을 검증한다.
- 파산채무가 있으면 새 레버리지 진입이 막히는지 검증한다.
- 일반 주식 매수/매도는 파산채무가 있어도 가능한지 검증한다.
- 골드 보상 지급 시 25%가 자동 상환되고 실제 수령액이 감소하는지 검증한다.
- 채무보다 상환액이 클 때 남은 채무가 0에서 멈추는지 검증한다.
- 기존 주식/카지노/RPG 테스트가 깨지지 않는지 전체 테스트를 돌린다.

## 밸런스 판정

이 설계는 100배 레버리지의 재미는 유지하면서, 실패 비용을 증거금 손실보다 약간 더 무겁게 만든다. 동시에 실제 마이너스 골드를 피하므로 통합 골드 시스템과 기존 게임 플레이를 크게 복잡하게 만들지 않는다.
