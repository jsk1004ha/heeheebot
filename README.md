# HeeHeeBot

디스코드 종합 봇의 초석입니다. 현재 구현된 기능은 **레벨 + 돈 시스템**입니다.

## 현재 포함된 기능

- 메시지 활동 보상
  - 일반 채팅마다 5~15 XP 지급
  - 하루 첫 채팅은 추가 50 XP 지급
  - 필요 경험치 공식: `floor(100 × 레벨^1.5)`
  - 경험치가 충분하면 자동 레벨업
  - 레벨업 보너스 지급
- `/프로필`
  - 내 레벨, 경험치, 연속 출석, 보유금 확인
- `/출석`
  - 하루 1회 100 XP + 500 코인 지급
  - 연속 출석 3/5/7/14/30일 배수에 추가 XP 지급
- `/송금`
  - 다른 유저에게 돈 송금
- `/랭킹`
  - 서버 내 레벨/경험치 기준 TOP 랭킹
- `/운세`
  - 한국시간 기준 오늘운세/어제운세 확인
  - 같은 서버, 같은 유저, 같은 날짜의 운세는 누가 보더라도 항상 동일
  - `대상` 옵션으로 다른 유저 운세 확인 가능
  - 운세 확인 보너스 XP는 하루 1회 지급
- 도박/미니게임
  - `/끝말잇기 시작`
    - 1분 동안 버튼으로 참가자를 모집한 뒤 시작
    - 모드 선택 가능: `일반`, `매너(한방단어 금지)`, `쿵쿵따(3글자만)`
    - 참가자와 희희봇 AI가 순서대로 단어를 입력
    - 자기 차례에 처음 30초 안에 답하지 못하면 탈락
    - 성공 단어가 나올 때마다 제한시간이 1초씩 줄어들며 최소 5초까지 감소
    - 최종 유저 승자는 보유금 보상 지급
    - 모든 참가자는 생존 순위에 따라 XP 지급(빨리 탈락할수록 적음)
    - 단어 DB는 AutoKkutu `KkutuDbDump/kor_list.txt` 기반 357,644개 한국어 단어(`data/wordchain-ko.txt`) 사용
  - `/홀짝 돈 선택`
    - 홀/짝 적중 시 약 1.9배 지급
    - 99~100은 하우스 승리라 실제 승률은 약 49%
  - `/주사위 돈 선택`
    - 높음(4~6) 또는 낮음(1~3) 적중 시 2배 지급
  - `/슬롯 돈`
    - 낮은 확률로 3배, 10배, 20배 지급
  - `/블랙잭 돈`
    - 상대를 비우면 AI와 대결
    - 승리 시 1.5배, 블랙잭 승리 시 2배, 무승부는 환불
  - `/블랙잭 돈 상대`
    - 상대 유저가 버튼으로 수락하면 시작
    - 양쪽 잔액을 검사한 뒤 같은 금액을 걸고 승자가 판돈 전체를 가져감
- 서버 관리 기능
  - `/청소 개수`
  - `/경고 유저 사유`
  - `/경고확인 유저`
  - `/경고삭제 유저`
  - `/뮤트 유저 시간 사유`
  - `/언뮤트 유저`
  - `/킥 유저 사유`
  - `/밴 유저 사유`
  - `/언밴 유저id`
  - `/관리설정 로그채널 채널`
  - `/관리설정 금칙어추가 단어`
  - `/관리설정 경고누적처벌 횟수 처벌 시간`
  - 자동 도배 대응
    - 기본값: 5초 안에 메시지 5개 또는 같은 메시지 3회 반복 시 먼저 10초 슬로우모드
    - 10분 안에 다시 도배로 적발되면 자동 밴
- SQLite 파일 저장소
  - 기본 경로: `data/profiles.sqlite`
  - 기존 `data/profiles.json`이 있으면 첫 실행 시 SQLite DB로 자동 마이그레이션
  - 추후 PostgreSQL 등 다른 DB 저장소로 교체하기 쉽게 서비스 계층 분리

## 시작하기

Node.js 22.5 이상이 필요합니다. SQLite 저장소는 Node.js 내장 `node:sqlite` 모듈을 사용합니다.

```bash
npm install
cp .env.example .env
```

`.env`에 봇 토큰과 애플리케이션 ID를 넣은 뒤:

```bash
set -a
source .env
set +a
npm run register
npm start
```

> 채팅 XP 자체는 메시지 이벤트만 있으면 동작하지만, 금칙어 감지와 자동 도배 대응은 메시지 내용을 확인합니다.
> Discord Developer Portal에서 **Message Content Intent**를 켜야 금칙어/도배 감지가 정상 동작합니다.

## 서버 관리 예시

```text
/관리설정 로그채널 #로그
/관리설정 금칙어추가 나쁜말
/관리설정 경고누적처벌 3 뮤트 10분
/경고 @유저 도배
/뮤트 @유저 10분 도배
/청소 30
```

봇 역할은 관리 대상 유저보다 위에 있어야 하며, 필요한 권한은 `메시지 관리`, `멤버 타임아웃`, `킥`, `밴`, `서버 관리`입니다.

## 도박 명령어 예시

```text
/끝말잇기 시작
/끝말잇기 시작 모드:매너(한방단어 금지)
/끝말잇기 시작 모드:쿵쿵따(3글자만)
/홀짝 100 홀
/주사위 100 높음
/슬롯 100
/블랙잭 500
/블랙잭 500 @상대
```

모든 도박 기능은 봇 내부 가상 돈만 사용합니다. 실제 돈 결제/환전 기능은 없습니다.
유저 간 블랙잭은 상대가 수락해야만 진행되며, 시작 전 양쪽 잔액을 검사합니다.

끝말잇기는 한 채널당 하나만 진행됩니다. 모집 메시지의 `참가` 버튼으로 참가하면 1분 뒤 희희봇 AI까지 포함한 순서로 시작됩니다. 매너 모드는 다음 사람이 이을 수 없는 한방단어를 금지하고, 쿵쿵따는 정확히 3글자 단어만 허용합니다.

## 운세 명령어 예시

```text
/운세
/운세 날짜:오늘운세
/운세 날짜:어제운세
/운세 대상:@유저
/운세 날짜:어제운세 대상:@유저
```

운세 결과는 사용자 ID와 날짜를 기준으로 고정됩니다. 그래서 다른 사람이 내 운세를 확인해도 내가 직접 보는 운세와 같은 문구가 나옵니다.

## 개발 명령

```bash
npm test
npm run register
npm start
```

`npm test`는 실제 Discord 테스트 계정 없이 가짜 유저 ID와 임시 SQLite DB만 사용합니다.
